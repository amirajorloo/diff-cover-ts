## Prerequisites

| Tool    | Minimum version | Notes                                                     |
| ------- | --------------- | --------------------------------------------------------- |
| Node.js | 20              | Required at runtime                                       |
| Bun     | 1.x             | Recommended for development (fast installs + test runner) |
| Git     | any             | Required for `diff-cover`/`diff-quality` to work          |

You can use `npm` or `yarn` instead of Bun, but the lockfile is `bun.lock` and scripts are tuned for Bun.

## Setting Up Your Development Environment

```bash
# 1. Clone the repository
git clone https://github.com/your-org/diff-cover-ts.git
cd diff-cover-ts

# 2. Install dependencies (also installs the Husky pre-commit hook via the prepare script)
bun install

# 3. Build
bun run build

# 4. Run tests to verify everything works
bun run test
```

For rapid iteration you can run the CLI directly from source via `tsx` (no build step):

```bash
# Run diff-cover directly
node --import tsx/esm src/cli/diff-cover.ts coverage/lcov.info

# Or, build first and use the dist output
bun run build
./dist/cli/diff-cover.js coverage/lcov.info
```

## Linting & Formatting

```bash
# Check for lint errors
bun run lint

# Auto-fix lint errors where possible
bun run lint:fix

# Check formatting (no writes)
bun run format:check

# Apply Prettier formatting to all files
bun run format
```

The **pre-commit hook** (powered by Husky + lint-staged) automatically runs ESLint and Prettier on staged files before every commit. It is installed automatically when you run `bun install` (via the `prepare` script).

If you need to bypass the hook in exceptional circumstances:

```bash
git commit --no-verify -m "your message"
```

## Project Structure

```
diff-cover-ts/
├── src/
│   ├── cli/           # CLI entry points (diff-cover, diff-quality) and config loader
│   ├── core/          # Orchestration logic (runDiffCover, runDiffQuality)
│   ├── coverage/      # Coverage format parsers (LCOV, Cobertura, Clover, JaCoCo)
│   ├── git/           # Git diff parsing (unified diff → FileDiff[])
│   ├── quality/       # Quality drivers (ESLint, Stylelint, Generic)
│   ├── reports/       # Output renderers (console, HTML, JSON, Markdown)
│   ├── __tests__/     # Vitest unit tests
│   ├── types.ts       # Shared TypeScript types / interfaces
│   ├── util.ts        # Shared utilities (path resolution, command runner, etc.)
│   └── index.ts       # Public API re-exports
├── templates/         # (Reserved) HTML report template assets
├── dist/              # Compiled output (generated by `bun run build`)
├── eslint.config.ts   # ESLint 9 flat config (loaded via jiti)
├── prettier.config.mjs # Prettier formatting config
├── lint-staged.config.mjs # lint-staged config (runs on pre-commit)
├── tsup.config.ts     # Build configuration
├── vitest.config.ts   # Test configuration
└── package.json
```

### Data flow for `diff-cover`

```
CLI args / config file
        │
        ▼
  resolveOptions()           ← src/cli/shared.ts
        │
        ▼
   runDiffCover()            ← src/core/diff-cover.ts
   ├── getChangedFiles()     ← src/git/diff-parser.ts   (git diff → FileDiff[])
   ├── parseCoverageFile()   ← src/coverage/index.ts    (file → CoverageData)
   └── mergeCoverageData()   ← src/coverage/index.ts    (merge multiple reports)
        │
        ▼
    ReportData
        │
        ▼
   writeReports()            ← src/reports/index.ts     (console / HTML / JSON / MD)
```

### Data flow for `diff-quality`

```
CLI args / config file
        │
        ▼
  resolveOptions()
        │
        ▼
  runDiffQuality()           ← src/core/diff-quality.ts
  ├── getChangedFiles()      ← src/git/diff-parser.ts
  ├── driver.run()           ← src/quality/<driver>.ts  (runs the linter)
  └── driver.parseReport()   ← src/quality/<driver>.ts  (parses output)
        │
        ▼
    ReportData  →  writeReports()
```

## Running Tests

```bash
# Run all tests once
bun run test

# Watch mode (re-runs on file save)
bun run test:watch
```

Tests live in `src/__tests__/` and are written with [Vitest](https://vitest.dev/). Each test file covers one module:

| Test file             | Module under test                  |
| --------------------- | ---------------------------------- |
| `coverage.test.ts`    | All coverage parsers + merge logic |
| `diff-parser.test.ts` | Git diff parsing                   |
| `quality.test.ts`     | Quality drivers                    |
| `reports.test.ts`     | Output renderers                   |
| `util.test.ts`        | Utility functions                  |

When adding new functionality, add corresponding tests in the appropriate file (or a new file if you're introducing a new module).

## Building

```bash
bun run build
```

This runs `tsup` and outputs compiled JS + `.d.ts` files to `dist/`. The CLI entry points (`dist/cli/diff-cover.js`, `dist/cli/diff-quality.js`) are automatically made executable.

## Adding a New Coverage Format

1. **Create a parser** in `src/coverage/<format>.ts`:

```typescript
import { resolveFilePath } from '../util.js'
import type { CoverageData } from './types.js'

/**
 * Parse <Format> coverage data.
 *
 * Brief description of the XML/text structure.
 */
export function parseMyFormat(content: string, gitRoot: string, srcRoots: string[]): CoverageData {
  const data: CoverageData = new Map()
  // ... parse content, populate data ...
  return data
}
```

`CoverageData` is `Map<filePath, { coveredLines: Set<number>, measuredLines: Set<number> }>`.
File paths must be git-relative (use `resolveFilePath()` from `src/util.ts`).

2. **Register the parser** in `src/coverage/index.ts` — add auto-detection logic in `parseCoverageFile()`.

3. **Update the README** — add the format to the "Supported Formats" table.

4. **Add tests** — add fixture data and assertions to `src/__tests__/coverage.test.ts`.

## Adding a New Quality Driver

1. **Create a driver** in `src/quality/<tool>.ts` implementing the `QualityDriver` interface:

```typescript
import type { Violation } from '../types.js'
import { executeCommand, toUnixPath } from '../util.js'
import type { QualityDriver } from './types.js'

export const myToolDriver: QualityDriver = {
  name: 'mytool',

  /**
   * Run the quality tool on the given files and return raw output.
   */
  async run(files: string[]): Promise<string> {
    const { stdout } = await executeCommand('npx', ['mytool', '--format=compact', ...files], [0, 1])
    return stdout

  /**
   * Parse tool output into a map of file path → violations.
   * Each line of output should match: file:line:col:message
   */
  parseReport(content: string): Map<string, Violation[]> {
    const result = new Map<string, Violation[]>()
    // ... parse content ...
    return result
  },
}
```

2. **Register the driver** in `src/quality/index.ts`:

```typescript
import { myToolDriver } from './mytool.js'

const drivers: Record<string, QualityDriver> = {
  eslint: eslintDriver,
  stylelint: stylelintDriver,
  mytool: myToolDriver, // ← add here
}
```

3. **Update the README** — add the tool to the "Quality tools" table.

4. **Add tests** — add fixture output and assertions to `src/__tests__/quality.test.ts`.

## Commit & PR Conventions

- **Commit messages**: Use the [Conventional Commits](https://www.conventionalcommits.org/) format:
  ```
  feat: add JUnit XML coverage parser
  fix: handle empty diff output gracefully
  docs: add CONTRIBUTING guide
  test: add clover parser edge cases
  ```
- **Branch names**: `feat/<short-description>`, `fix/<short-description>`, `docs/<short-description>`
- **Pull requests**: Keep PRs focused — one feature or fix per PR.
- **Tests**: All new code must have corresponding tests. CI will run `bun run test` on every PR.
- **Build**: Run `bun run build` before opening a PR to ensure the TypeScript compiles cleanly.

If you're fixing a bug, please include a test that reproduces the bug before your fix.
